Fire Detection System with AI

Project Overview

This project focuses on developing a fire detection system using embedded components to automate the detection of fire hazards in specific areas. The system leverages sensors and real-time image analysis to quickly identify fire presence, thereby reducing reaction time in emergencies. It is designed to immediately trigger alerts when a fire is detected to minimize damage and inform the relevant individuals and systems. The system also offers the capability to store video clips of incidents and send notifications via SMS and email.

Key Features

  • Fire Detection: Utilizes appropriate sensors (e.g., gas sensors, temperature sensors) to detect fire indicators such as flammable gases or abnormal temperature increases.
  • Visual Detection: A camera captures images that are analyzed in real-time using AI-based fire detection models.
  • Reactions to Fire Detection: When fire is detected, the system activates alarms (sound, LED light, water vapor) and sends alerts via SMS and email, including video clips of the incident.
  • Data Storage: Incident video clips, along with timestamps and sensor data, are stored for future reference.
  • Remote Notifications: Automatic alerts are sent through SMS and email, including video clips, ensuring immediate response.
  • Image Processing: The system allows for image processing either locally (on the embedded system) or externally for better speed and alert efficiency.

Technologies Used

  • Raspberry Pi - Used as the main embedded platform for running the system and handling data processing.
  • Camera - Captures real-time images for fire detection using AI models.
  • DHT11 - Used for temperature and humidity measurement to detect abnormal conditions that may indicate fire.
  • MQ2 - A gas sensor used for detecting flammable gases, which can indicate the presence of fire or smoke.
  • Buzzer - Used for sound alerts when fire is detected.
  • LED - Visual alert (e.g., lights up when fire is detected).
  • AI Model in TFLite - A TensorFlow Lite model used for real-time fire detection and image classification.
  • Flask - A web framework used to create an interface for controlling and monitoring the system.
  • Streamlit - Used to create a user-friendly web dashboard to display system status and alerts.
  • Infobip - Provides SMS and email notification services to alert users remotely when a fire is detected.

Smart Farming System

Project Overview

A smart farming system leveraging IoT and AI to automate irrigation based on environmental sensors and detect plant diseases using AI, aimed at optimizing resource use and enhancing crop health.

Key Features

  • Real-time soil moisture, temperature, and humidity monitoring.
  • The system keeps track of component health and automatically detects any sensor malfunctions.
  • Automated irrigation
  • Plant disease detection via CNN model trained on the PlantVillage dataset
  • Web dashboard for data visualization, irrigation history, and notifications.
  • The continuous collection and storage of data
  • Email alerts for disease detection and system events.

Technologies Used

Hardware Components

  • DHT11 (Temperature & Humidity Sensor)
  • Soil Moisture Sensor
  • ESP32CAM (Camera Module)
  • ESP32WROOM (Main Controller)
  • Water Pump System

Software Stack

  • Python & Flask (Backend)
  • React JS (Frontend)
  • NodeRed (Flow Programming)
  • TensorFlow (CNN Models)
  • MySQL Database

Communication

  • Wi-Fi (Local Network)
  • MQTT Protocol

Intelligent Attendance Management and Liveliness Detection System

Project Overview

During our academic experience, we observed that manual attendance taking using paper sheets leads to a significant time loss and is prone to fraud, where one student signs in for another. To solve this problem, we developed an intelligent system that combines embedded components and AI algorithms to ensure automated and secure attendance management.

Key Features

  • Full Automation – Eliminates manual attendance sheets and saves time.
  • Enhanced Security – Prevents fraud using facial recognition and anti-spoofing techniques.
  • Real-Time Monitoring – Can be integrated with a web interface for attendance tracking.
  • Time Tracking – Logs entry and exit timestamps to monitor attendance duration.
  • Web API Communication – Ensures seamless data transfer between ESP32 and the server.
  • User-Friendly System – Compact, efficient, and functional both locally and online.

Technologies Used

  • ESP32-CAM – Microcontroller with an integrated camera for facial recognition.
  • PIR Sensor – Motion detection to optimize image capture.
  • MTCNN + FaceNet – Face detection and feature extraction.
  • SVM (Support Vector Machine) – Classification model for identifying known faces.
  • Haar Cascade + Anti-Spoofing Model – Eye blink detection and spoofing prevention for printed or replay attacks.
  • Flask API – Web service for sending captured images to a server for processing.

System Workflow:

  • Presence Detection using the PIR sensor, which triggers the ESP32-CAM.
  • Face Detection via MTCNN once the image is captured.
  • Feature Extraction using FaceNet for facial embeddings.
  • Identity Verification through a pre-trained SVM model.
  • Spoofing Attack Prevention:Eye blink detection with Haar Cascade + Anti-spoofing model to detect printed or replay attacks
  • Attendance Validation:The system records the entry and exit times of each individual.
  • Flask API Integration:ESP32-CAM sends images to a Flask server via HTTP requests + The server processes the image, verifies the identity, and logs attendance.
  • Real-Time Notifications: If unauthorized access or spoofing is detected, an instant notification is sent to the responsible person (e.g., instructor, administrator) , A notification with the entry or exit timestamp is also sent upon attendance verification.
